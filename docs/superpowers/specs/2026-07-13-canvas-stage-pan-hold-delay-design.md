# Studio 表盘内部整体移动长按时长设计

## 背景

Studio 已支持整体移动画布舞台：表盘轮廓外按下后立即拖动；表盘轮廓内长按达到阈值后，将指针控制权从 Fabric 元素编辑交给画布舞台移动。当前表盘内部的长按阈值为 400ms，容易在编辑元素时误触整体移动。

## 目标

- 圆形表盘内部需要持续按住 1500ms，才解锁整体画布移动。
- 矩形表盘内部同样需要持续按住 1500ms，才解锁整体画布移动。
- 表盘轮廓外仍保持按下立即拖动。
- 长按期间指针移动超过现有 6px 容差时，取消本次解锁。

## 非目标

- 不改变表盘轮廓命中检测。
- 不改变元素选择、移动、缩放或旋转行为。
- 不改变 pointer owner、Fabric 指针交接、边界限制或三层同步移动逻辑。
- 不增加设置项，也不将长按时长写入设计数据。

## 方案

保留现有 `immediate` 与 `long-press` 区域状态机，将 `canvasPan.ts` 中统一的表盘内部长按延迟常量由 400ms 调整为 1500ms。`Design.vue` 继续读取该常量启动解锁定时器，不新增圆形和矩形分支。

这样能让圆形和矩形表盘内部保持完全一致的交互契约，同时不扩大现有指针生命周期和清理逻辑。

## 交互流程

1. 指针在表盘轮廓外按下时，立即进入整体画布移动。
2. 指针在圆形或矩形表盘轮廓内按下时，启动 1500ms 定时器。
3. 1500ms 内若指针移动超过起点 6px，取消定时器，事件继续按原有元素编辑流程处理。
4. 指针保持在容差内达到 1500ms 后，完成 Fabric 交接并解锁整体画布移动。
5. `pointerup`、`pointercancel` 或组件卸载继续复用现有清理流程。

## 测试与验收

- 单元测试断言表盘内部长按延迟为 1500ms。
- 单元测试继续覆盖 6px 移动容差及其边界。
- 圆形表盘内部在 1500ms 前不会解锁，达到 1500ms 后可以整体移动。
- 矩形表盘内部在 1500ms 前不会解锁，达到 1500ms 后可以整体移动。
- 圆形和矩形表盘轮廓外均保持即时整体移动。
- 运行画布平移专项测试、类型检查和生产构建。
