# 画布图层顺序控制点设计

## 背景

Wristo Studio 已支持在左侧图层面板拖拽调整元素顺序，但用户在画布中选中元素后，无法集中完成克隆、删除和图层排序操作。本功能在元素选框右下角增加一个统一操作入口，让用户不离开画布即可管理当前元素。

## 目标

- 在不取代现有右下角等比缩放点的前提下，增加图层顺序控制入口。
- 支持置顶、上移一层、下移一层和置底四种操作。
- 操作结果立即同步画布、左侧图层列表、撤销历史和持久化配置。
- 复用现有图层管理能力，保持拖拽排序与控制点排序的行为一致。
- 将克隆、删除和图层顺序操作统一收纳到右下角操作入口。

## 非目标

- 不建立固定的“装饰层、信息层、前景层”等业务分组。
- 不改变左侧图层面板的拖拽排序交互。
- 不允许移动 `global`、`background` 等固定层。
- 不调整编辑器 DOM 浮层的 CSS `z-index`。

## 交互设计

### 统一操作入口

在元素右下角缩放点外侧增加一个绿色统一操作控制点。入口相对右下角端点的横向、纵向偏移均为 `10px`。入口使用能够表达对象操作的图标，并提供 `Object Actions` tooltip。

移除左上角红色删除控制点和右上角绿色克隆控制点。克隆与删除能力仅保留在统一操作菜单中，避免同一操作出现两个画布入口。

仅当选中对象满足以下条件时显示入口：

- 对象是可管理的普通表盘元素；
- 对象不是 `global` 或 `background`；
- 对象未锁定，且允许选择和交互；
- 对象启用了控制点。

### 展开菜单

点击统一操作入口后，在元素右侧向上展开六个胶囊形操作项，避免遮挡元素下方区域。每项同时绘制图标和英文文字，不再依赖用户识别纯图标。菜单从上到下排列为：

1. `Clone`：克隆当前元素并选中新元素；
2. `Delete`：删除当前元素；
3. `Bring to Front`：置于所有可排序元素之上；
4. `Bring Forward`：向上移动一个可排序层级；
5. `Send Backward`：向下移动一个可排序层级；
6. `Send to Back`：置于固定背景层之上的最低可排序位置。

胶囊按钮采用统一宽度和高度，图标位于左侧、英文文字位于右侧。`Delete` 使用红色图标或文字，其余操作使用绿色主色；禁用的图层操作使用灰色。触摸命中区域覆盖完整胶囊，且相邻操作项不得重叠。

再次点击入口、点击画布空白区域、切换选中元素或取消选择时，菜单自动收起。执行图层排序后，菜单保持展开且元素保持选中；执行克隆后菜单切换到新克隆元素并收起；执行删除后菜单随元素一起消失。

当元素已位于最上方时，禁用 `Bring to Front` 和 `Bring Forward`；当元素已位于最下方时，禁用 `Send Backward` 和 `Send to Back`。禁用按钮仍可见，但采用弱化样式且不响应操作。

## 技术设计

### 控制点管理

在 `src/utils/controlManager.ts` 的统一控制点体系中维护一个绿色操作入口和六个菜单操作项。继续保留现有 `br` 等比缩放控制点；统一入口使用独立 control key 和外偏移位置。删除原有 `cloneControl`、`deleteControl` 左右上角控制点定义，但复用其处理函数和回调。

展开状态按当前选中对象维护，不写入设计配置。状态至少包含当前展开对象 ID；选择变化或画布取消选择时清空。菜单控制点的可见性由展开状态与对象当前层级共同决定。

入口及胶囊菜单采用 Fabric 控制点绘制与命中机制，不引入需要跟随缩放、旋转和视口变化重新定位的 DOM 浮动菜单。

### 图层操作

控制点处理函数只负责解析目标对象并调用 `src/engine/managers/layerManager.ts` 中已有的：

- `bringToFront(id)`；
- `bringForward(id)`；
- `sendBackward(id)`；
- `sendToBack(id)`。

固定层规则继续由 `layerManager.ts` 统一维护。控制点不得复制一套对象移动算法。

每次成功调整顺序后：

1. `layerManager` 更新 Fabric 对象顺序并同步左侧图层列表；
2. 保持目标元素为当前活动对象；
3. 记录一次可撤销的 `layer:reorder` 历史状态，并捕获最新配置；
4. 配置生成流程从画布顺序刷新 `orderIds`，沿用现有保存与重新加载机制。

若操作不会改变顺序，例如已在最上层时再次置顶，则不创建无意义的历史记录。

### 边界情况

- 元素在菜单展开期间被删除：立即清除展开状态，不再执行操作。
- 元素在菜单展开期间被锁定或变为不可交互：收起菜单。
- 多选：首版不显示统一操作入口，避免多对象克隆、删除和相对顺序语义不明确。
- 组合元素：将注册到图层系统的主元素作为单个层级移动，不改变其内部子对象顺序。
- 画布旋转、缩放或平移：控制点继续由 Fabric 跟随活动对象绘制。

## 可访问性与文案

统一入口需要有 `Object Actions` tooltip。展开菜单直接显示 `Clone`、`Delete`、`Bring to Front`、`Bring Forward`、`Send Backward`、`Send to Back`，同时保留对应 tooltip。各胶囊之间应保持足够命中间距，触摸命中区域不得互相覆盖。

## 测试与验收

### 自动验证

- 为图层位置判断和无变化操作补充可独立测试的逻辑测试（若当前测试设施支持）。
- 运行 `npm run build`，验证 TypeScript 与生产构建。
- 运行 `git diff --check`。

### 手动验收

- 普通元素右下角保留缩放点，并在外侧显示绿色统一操作入口。
- 左上角不再显示删除点，右上角不再显示克隆点。
- 点击入口后，六个带英文文字的胶囊操作项按约定顺序向上展开；再次点击或取消选择后收起。
- `Clone` 创建并选中新元素，`Delete` 删除当前元素，均继续支持撤销历史。
- 四种操作均立即改变画布遮挡关系和左侧图层顺序。
- 顶层和底层对应按钮正确禁用，且无效点击不产生撤销记录。
- 排序后可撤销、重做，保存并重新进入设计后顺序不变。
- 背景、全局、锁定元素和多选状态不显示统一操作入口。
- 缩放、克隆、删除以及元素专属控制点仍可正常使用。

## 实施范围

改动限定在 `wristo-studio`。预计涉及统一控制点管理、画布选择状态监听、图层历史记录接入、英文文案及相关测试，不修改其他 Wristo 仓库。
