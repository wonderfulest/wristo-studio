# 画布图层顺序控制点设计

## 背景

Wristo Studio 已支持在左侧图层面板拖拽调整元素顺序，但用户在画布中选中元素后，无法直接完成置顶、上移、下移或置底操作。本功能在元素选框右下角增加一个独立的图层入口，让用户不离开画布即可调整当前元素的堆叠顺序。

## 目标

- 在不取代现有右下角等比缩放点的前提下，增加图层顺序控制入口。
- 支持置顶、上移一层、下移一层和置底四种操作。
- 操作结果立即同步画布、左侧图层列表、撤销历史和持久化配置。
- 复用现有图层管理能力，保持拖拽排序与控制点排序的行为一致。

## 非目标

- 不建立固定的“装饰层、信息层、前景层”等业务分组。
- 不改变左侧图层面板的拖拽排序交互。
- 不允许移动 `global`、`background` 等固定层。
- 不调整编辑器 DOM 浮层的 CSS `z-index`。

## 交互设计

### 图层入口

在元素右下角缩放点外侧增加一个独立的图层控制点，与缩放点保持约 10px 的视觉间距。入口使用能够表达图层堆叠的图标，并提供 `Layer Order` tooltip。

仅当选中对象满足以下条件时显示入口：

- 对象是可管理的普通表盘元素；
- 对象不是 `global` 或 `background`；
- 对象未锁定，且允许选择和交互；
- 对象启用了控制点。

### 展开菜单

点击图层入口后，在元素右侧向上展开四个控制按钮，避免遮挡元素下方区域。按钮从上到下排列为：

1. `Bring to Front`：置于所有可排序元素之上；
2. `Bring Forward`：向上移动一个可排序层级；
3. `Send Backward`：向下移动一个可排序层级；
4. `Send to Back`：置于固定背景层之上的最低可排序位置。

再次点击入口、点击画布空白区域、切换选中元素或取消选择时，菜单自动收起。执行任一排序操作后，菜单保持展开，元素保持选中，方便连续调整。

当元素已位于最上方时，禁用 `Bring to Front` 和 `Bring Forward`；当元素已位于最下方时，禁用 `Send Backward` 和 `Send to Back`。禁用按钮仍可见，但采用弱化样式且不响应操作。

## 技术设计

### 控制点管理

在 `src/utils/controlManager.ts` 的统一控制点体系中增加图层入口和四个菜单控制点。继续保留现有 `br` 等比缩放控制点；图层入口使用独立 control key 和外偏移位置。

展开状态按当前选中对象维护，不写入设计配置。状态至少包含当前展开对象 ID；选择变化或画布取消选择时清空。菜单控制点的可见性由展开状态与对象当前层级共同决定。

入口及菜单采用 Fabric 控制点绘制与命中机制，不引入需要跟随缩放、旋转和视口变化重新定位的 DOM 浮动菜单。

### 图层操作

控制点处理函数只负责解析目标对象并调用 `src/engine/managers/layerManager.ts` 中已有的：

- `bringToFront(id)`；
- `bringForward(id)`；
- `sendBackward(id)`；
- `sendToBack(id)`。

固定层规则继续由 `layerManager.ts` 统一维护。控制点不得复制一套对象移动算法。

每次成功调整顺序后：

1. `layerManager` 更新 Fabric 对象顺序并同步左侧图层列表；
2. 保持目标元素为当前活动对象；
3. 记录一次可撤销的 `layer:reorder` 历史状态，并捕获最新配置；
4. 配置生成流程从画布顺序刷新 `orderIds`，沿用现有保存与重新加载机制。

若操作不会改变顺序，例如已在最上层时再次置顶，则不创建无意义的历史记录。

### 边界情况

- 元素在菜单展开期间被删除：立即清除展开状态，不再执行操作。
- 元素在菜单展开期间被锁定或变为不可交互：收起菜单。
- 多选：首版不显示图层入口，避免多对象相对顺序语义不明确。
- 组合元素：将注册到图层系统的主元素作为单个层级移动，不改变其内部子对象顺序。
- 画布旋转、缩放或平移：控制点继续由 Fabric 跟随活动对象绘制。

## 可访问性与文案

控制点需要有明确 tooltip：`Layer Order`、`Bring to Front`、`Bring Forward`、`Send Backward`、`Send to Back`。图标之间应保持足够命中间距，触摸命中区域沿用或大于现有控制点的 `touchCornerSize`。

## 测试与验收

### 自动验证

- 为图层位置判断和无变化操作补充可独立测试的逻辑测试（若当前测试设施支持）。
- 运行 `npm run build`，验证 TypeScript 与生产构建。
- 运行 `git diff --check`。

### 手动验收

- 普通元素右下角保留缩放点，并在外侧显示图层入口。
- 点击入口后，四个按钮按约定顺序向上展开；再次点击或取消选择后收起。
- 四种操作均立即改变画布遮挡关系和左侧图层顺序。
- 顶层和底层对应按钮正确禁用，且无效点击不产生撤销记录。
- 排序后可撤销、重做，保存并重新进入设计后顺序不变。
- 背景、全局、锁定元素和多选状态不显示图层入口。
- 缩放、克隆、删除以及元素专属控制点仍可正常使用。

## 实施范围

改动限定在 `wristo-studio`。预计涉及统一控制点管理、画布选择状态监听、图层历史记录接入、英文文案及相关测试，不修改其他 Wristo 仓库。
