# GoalBar 渐变前景色设计

## 目标

让 `goalBar` 与 `goalArc` 一样支持双色渐变前景色，并保证 Studio 预览、设计导出、Connect IQ 代码生成和真机渲染结果一致。

## 配置契约

`goalBar` 复用 `goalArc` 的现有字段：

- `gradientEnabled`: 是否启用渐变。
- `gradientStartColor`: 进度起点颜色。
- `gradientEndColor`: 进度终点颜色。

未启用渐变、颜色缺失或颜色无效时，继续使用现有 `color` 单色前景，不改变旧设计的行为。

## Studio

GoalBar 设置面板的前景色选择器复用 GoalArc 的渐变交互。配置修改通过现有元素更新链路同步到 Fabric 对象、元素存储快照和导出配置。

GoalBar 画布渲染器根据进度方向绘制渐变：

- 左对齐：左侧为起始色，沿进度方向向右过渡到结束色。
- 右对齐：右侧为起始色，沿进度方向向左过渡到结束色。
- 连续、分段、自定义多边形三种渲染方式遵循同一方向语义。

## Connect IQ 生成与运行时

脚手架提取器沿用 GoalArc 的颜色规范化规则，把三个渐变字段传入生成上下文。`SuperAlphaView.j2.mc` 将字段写入 `GoalBar.draw` 的选项字典。

`GoalBar.mc` 使用有限分片实现 Garmin Canvas 上的渐变：

- 连续条在已完成的前景区域内按进度方向分片并插值颜色。
- 分段条按每个可见段在整体已完成进度中的位置取色，保持段间空隙。
- 自定义多边形先按进度裁剪，再沿进度方向分片裁剪和填色。
- 分片数设置上限，避免高分辨率或复杂多边形造成不受控的绘制开销。

背景色、边框、内边距、圆角及原有进度裁剪行为保持不变。

## 测试与验收

按测试驱动方式先补失败用例，再实现：

- Studio 编解码可往返保存渐变字段，并兼容无字段的旧配置。
- Studio 渲染器在左右进度方向下使用正确的颜色顺序。
- 脚手架可规范化并输出 GoalBar 渐变字段，无效颜色回退到前景色。
- Jinja 模板把渐变选项传给 `GoalBar.draw`。
- Monkey C 运行时覆盖单色回退、左右方向、连续、分段和自定义多边形路径。

最终运行相关单元测试、`wristo-studio` 构建、脚手架测试，并执行可用的 Connect IQ 构建验证。

## 非目标

- 不为背景色或边框增加渐变。
- 不增加三色以上的渐变色标。
- 不改变 GoalArc 的既有行为。
