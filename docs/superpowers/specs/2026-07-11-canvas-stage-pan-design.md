# Studio 表盘外部拖动画布设计

## 背景

Studio 当前使用 Fabric 6 渲染可编辑元素，并通过 Vue DOM 层渲染黑色表盘底层和设备外框。`Canvas.vue` 已有缩放状态和视口偏移入口，`canvasManager.ts` 也保留了空格键光标状态，但目前没有完整的画布平移事件。

如果仅修改 Fabric 的 `viewportTransform`，Fabric 内容会移动，但独立的表盘底层和设备外框不会自动移动，容易产生视觉错位。因此，本功能采用整个画布舞台的 DOM 平移。

## 目标

- 鼠标左键或主触点在表盘实际轮廓外按下后，立即进入画布拖动。
- Fabric 内容、黑色表盘底层和设备外框作为一个整体移动。
- 表盘内部原有的元素选择、框选、移动、缩放和旋转交互保持不变。
- 画布平移只属于编辑器视图状态，不写入设计配置、自动保存或撤销历史。
- 平移后，标尺、素材拖放和 Fabric 指针坐标仍然正确。
- 画布不能被完全拖出工作区，始终保留可重新抓取的区域。

## 非目标

- 不增加空格键加拖动、鼠标中键拖动或触控双指缩放。
- 不持久化平移位置，重新进入设计页时仍从居中位置开始。
- 不改变现有缩放倍率、缩放按钮或设计元素坐标。
- 不修改任何表盘业务配置或 Connect IQ 导出格式。

## 方案选择

### 采用：整体 DOM 舞台平移

在 `Design.vue` 中将承载 `CanvasView` 的外层容器明确命名为 `canvas-stage`，用响应式 `panOffset` 生成 `translate3d(x, y, 0)`。舞台内部的 Fabric Canvas、表盘底层和设备外框自然保持相对对齐。

优点：

- 不改变 Fabric 世界坐标和对象状态。
- 设备外框、背景和画布内容无需分别同步偏移。
- 与当前缩放实现相互独立，缩放负责大小，舞台平移负责位置。
- 不会触发 Fabric 的对象修改历史。

### 未采用：Fabric viewport 平移

现有 `canvasOffset` 可以写入 `viewportTransform[4]` 和 `[5]`，但还需要同步修改表盘底层、设备外框、裁剪区域和标尺。多个渲染层需要共享同一偏移，耦合较高。

### 未采用：原生滚动容器

滚动容器会改变当前画布居中模型，并使缩放、标尺与工作区尺寸变化的组合行为更复杂，不适合本次交互。

## 组件职责

### `Design.vue`

- 持有工作区引用、舞台引用和瞬时 `panOffset`。
- 在 `.center-area` 的捕获阶段处理 Pointer Events。
- 完成表盘轮廓命中检测、拖动生命周期、边界限制和光标状态。
- 在平移帧完成后请求标尺刷新，并同步 Fabric DOM 偏移。

### `Canvas.vue`

- 继续管理 Fabric 初始化、缩放、表盘底层和设备外框。
- 暴露一个轻量的 `syncCanvasOffset()` 方法，内部调用 Fabric `calcOffset()`。
- 不把舞台平移值写进 Fabric `viewportTransform`。

### `CanvasRulers.vue`

- 暴露 `refresh()` 方法，复用现有 `update()` 绘制逻辑。
- 根据 `canvas-stage` 的实际屏幕位置重新计算刻度原点。
- 标尺自身固定在工作区，不随舞台 DOM 一起移动。

### `canvasManager.ts`

- 删除只改变光标、但没有实际拖动能力的空格键状态。
- 删除通过全局 `.canvas-container` 选择器初始化外层容器样式的逻辑。
- 避免 Fabric 自动生成的 `.canvas-container` 与 Studio 舞台容器同名造成歧义。

## 交互流程

1. `.center-area` 在捕获阶段收到主指针的 `pointerdown`。
2. 忽略非左键、非主触点、缩放/历史控件、时间模拟器和标尺等交互区域。
3. 取得 Fabric 上层画布的屏幕矩形，并判断指针是否位于表盘实际轮廓外：
   - 圆形设计：指针到表盘圆心的距离大于屏幕半径时，属于外部区域。
   - 矩形设计：指针不在表盘屏幕矩形内时，属于外部区域。
4. 如果位于表盘内部，事件继续交给 Fabric，原有编辑行为不变。
5. 如果位于表盘外部，清除当前元素选择、记录起始指针和起始偏移，并调用 `setPointerCapture()`。
6. `pointermove` 根据屏幕坐标差更新目标偏移，在 `requestAnimationFrame` 中统一应用 CSS transform。
7. 每帧对偏移进行限制，确保舞台至少有 64px 留在可见工作区内。
8. `pointerup`、`pointercancel` 或组件卸载时结束拖动并释放 pointer capture。
9. 平移更新后刷新标尺；拖动结束后调用 Fabric `calcOffset()`，确保后续点击和素材拖放坐标正确。

## 命中检测与坐标

- 命中检测使用 `getBoundingClientRect()` 的屏幕坐标，不读取 Fabric 对象坐标，因此天然包含当前缩放和舞台平移。
- 圆形表盘使用 `min(rect.width, rect.height) / 2` 作为屏幕半径。
- 设备外框可能超出表盘，但它是 `pointer-events: none`，不会抢占拖动事件。
- CSS transform 只作用于 `canvas-stage`；Fabric 内部缩放仍由现有 `zoomManager` 管理。

## 边界限制

边界限制以工作区内容矩形和舞台未平移矩形为基础计算。允许大尺寸或高倍率画布超出工作区，但在水平和垂直方向上至少保留 64px 可见区域。工作区或侧边栏尺寸变化后，重新限制当前偏移，避免画布因布局变化而完全不可见。

## 状态与历史

`panOffset`、拖动起点、活动 pointer ID 和拖动标记均为 `Design.vue` 的瞬时状态。它们不进入 Pinia 持久化、设计配置、自动保存或 Fabric 历史。拖动本身不会产生 `object:modified`。

## 清理与异常处理

- 组件卸载时取消未执行的动画帧并移除窗口尺寸监听。
- `pointercancel` 与 `pointerup` 使用同一结束流程。
- 如果 pointer capture 已丢失，结束流程仍可安全重复调用。
- 如果 Fabric 尚未初始化，只允许移动舞台，不调用 `calcOffset()`。

## 验收与验证

- 在圆形表盘内部拖动元素，行为与修改前一致。
- 在圆形表盘四角透明区域及工作区空白处按下，画布立即跟随拖动。
- 方形表盘只在矩形外部区域触发画布拖动。
- 黑色表盘底层、Fabric 内容和设备外框始终同步移动。
- 指针移出工作区后松开，拖动能正常结束。
- 画布无法被完全拖出可见区域。
- 先缩放再拖动、先拖动再缩放，位置和比例均正确。
- 开启标尺后拖动画布，刻度原点同步更新，标尺本体保持固定。
- 平移后拖入 SVG/PNG，新增元素落点正确。
- 平移不新增撤销历史，保存和导出结果不包含视图偏移。
- 运行 `npm run build` 验证 TypeScript 和生产构建。
