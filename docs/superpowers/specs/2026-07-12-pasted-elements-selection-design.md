# 粘贴元素组保持选中设计

## 目标

当用户在 Studio 中同时选中多个元素并复制、粘贴后，取消原元素选区，仅选中本次成功粘贴产生的全部新元素。

## 范围

- 修改 `clipboardManager.ts` 的粘贴完成逻辑。
- 不改变复制内容、粘贴位置递增偏移或历史记录语义。
- 不改变单个元素复制粘贴的既有交互，但单个新元素仍应成为当前选中对象。

## 设计

`pasteSelection()` 在现有无历史记录事务中逐个创建元素，并收集每次 `addElement` 成功后对应的 Fabric 对象。事务完成后统一替换画布选区：

- 成功创建多个对象时，使用 Fabric `ActiveSelection` 将它们组成当前选区，并保持多选控制点禁用的现有约定。
- 只成功创建一个对象时，直接将该对象设为当前活动对象。
- 没有对象创建成功时，不创建空选区。

选区设置完成后统一请求画布重绘。Fabric 选择事件继续负责同步 `canvasStore` 中的活动元素 ID；若测试或运行时发现程序化设置不会稳定触发相应事件，则在同一选区提交入口显式同步 ID，避免画布选区与应用状态分离。

## 失败处理

单个元素粘贴失败仍沿用当前的告警与继续处理策略。最终选区只包含成功创建的对象，不包含失败项，也不恢复原选区。

## 历史记录

所有元素仍在一次 `runWithoutRecording` 中创建。完成选区更新后仅调用一次 `saveState('paste:selection')`，避免选区构造产生额外历史节点。

## 测试

- 多个元素全部粘贴成功：原选区被替换，新对象组成一个活动多选区。
- 多个元素部分粘贴失败：活动选区只包含成功创建的对象。
- 只粘贴成功一个元素：该元素成为当前活动对象。
