# 矩形渐变透明度修复设计

## 问题

Connect IQ 的矩形渐变由一组逐渐缩小、互相覆盖的矩形组成。每个矩形虽然都应用了 `opacity`，但同一像素会被重复 Alpha 合成；当透明度为 `0.2` 时，靠近渐变起点的区域最终接近不透明，与 Studio 的单层 Fabric 渐变不一致。

## 方案

保留现有颜色插值、四方向、透明度转换和独立边框路径，只替换渐变切片几何：

- 将渐变区域划分为互不重叠的条带，每个像素只填充一次。
- 水平渐变使用竖向条带，垂直渐变使用横向条带。
- 每条带在圆角区域根据圆方程计算内缩量，使条带组合保持原矩形圆角轮廓。
- 条带颜色取条带中心在完整渐变轴上的位置，反向渐变继续反转颜色比例。
- `opacity` 仍通过 `ColorUtils.colorWithOpacity()` 应用于每条带；边框保持单独绘制一次。

不修改 Studio 数据模型、面板、编码器、提取脚本或 SuperAlpha 模板。

## 兼容与边界

- `borderRadius <= 0` 时条带覆盖完整矩形。
- 圆角半径限制为不超过宽高的一半。
- 渐变方向和非法配置的现有回退行为保持不变。
- 切片数量继续受 `MAX_GRADIENT_SLICES` 限制，避免增加设备绘制负担。
- 纯色填充和边框路径不变。

## 验证

- 契约测试要求存在非重叠条带边界计算，并禁止旧的嵌套尺寸比例算法。
- 运行矩形渐变专项测试和相关 scaffold 测试。
- 运行 `git diff --check`。
- 若当前环境无法运行 Connect IQ 模拟器，只声明代码与测试验证，不声明视觉验证。
