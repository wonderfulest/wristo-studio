# Studio 菜单快捷添加智能落位

## 背景

Studio 顶部菜单通过 `AppMenu.vue` 快捷添加元素时，当前会把缺少坐标的普通元素统一放到设计中心。时间、日期、状态、天气、数据字段、图表和形状连续添加后容易相互覆盖，用户必须逐个拖开。

本设计为顶部菜单快捷添加引入“类型优先区 + 自动避让”。落位既考虑元素语义，也考虑当前画布已有内容。时针、分针、秒针和中心盖等轴心元素仍固定在设计中心。

## 目标

- 普通元素不再统一添加到画布中心。
- 根据元素类型优先选择符合表盘布局习惯的区域。
- 使用现有 Fabric 元素的实际位置和边界避让已有内容。
- 将数据字段和 Goal 等多元素快捷项作为一个完整布局块落位。
- 画布拥挤时仍完成添加，并选择安全区内重叠面积最小的位置。
- 同样的画布状态和添加类型产生稳定、可预测的结果。
- 最终坐标进入现有保存、导出、撤销和重新加载链路。

## 范围

本次只修改顶部菜单栏的快捷添加流程及其专用落位逻辑。

不包含：

- 左侧 Add Element 面板的添加行为。
- 复制、粘贴、导入和重新加载时的坐标。
- 用户手动拖拽、对齐和图层排序行为。
- 自动重排画布上已经存在的元素。
- 根据内容自动缩放元素。

现有字体、颜色、格式、数据绑定、显示状态和图层顺序语义保持不变。

## 已确认的布局规则

### 轴心元素

以下元素绕过智能落位，继续使用 `designSpec.centerX` 和 `designSpec.centerY`：

- `hourHand`
- `minuteHand`
- `secondHand`
- `centerCap`

已有的模拟结构元素 `global`、`background`、`tick12`、`tick60`、`romans`、上述指针和中心盖不作为矩形障碍参与碰撞评分。指针会旋转，刻度覆盖整个表盘，若把它们的 Fabric 外接矩形当作普通占用区，会错误地排斥所有内容元素。

### 普通元素类型优先区

候选锚点使用相对画布尺寸的坐标，不能依赖固定的 `454 × 454` 画布：

| 类型 | 首选区域 | 后续区域 |
| --- | --- | --- |
| 时间 | 上部居中 | 上部左右 |
| 日期 | 已有时间元素下方 | 上部偏中、上部左右 |
| 状态与通知图标 | 顶部边缘 | 左右边缘 |
| 天气 | 上部左右 | 中部左右 |
| 数据字段组合 | 中下部居中 | 中下部左右、底部 |
| 图表、Goal Bar | 下部居中 | 中下部左右 |
| Goal Arc | 中部左右 | 中下部居中或左右 |
| 图片、矩形、圆、直线 | 通用空白区 | 从中心向四周扩展 |

实现中的类型配置表保存候选区域的优先顺序。每个区域先生成首选锚点，再以标准 `454 × 454` 画布上的 12 像素为基准步长，在锚点附近按上、右、下、左和四个对角方向逐圈扩展；实际步长乘以 `min(width, height) / 454`。这些候选仍不合适时，使用相同步长、覆盖整个圆形安全区的固定顺序网格作为兜底。候选生成顺序固定，避免相同输入下的位置随机变化。

### 组合元素

以下快捷项按一个完整布局块处理：

- 数据字段：图标、数值和可选单位。
- Goal Bar：进度条、图标和数值。
- Goal Arc：圆弧、图标和数值。

菜单预设坐标只定义组合内部的相对位置，不再代表最终画布绝对坐标。落位管理器计算所有成员的联合边界，选择一个位置后给每个成员应用相同位移。真实边界校验和后续微调也只能整体平移，不能改变组合内部间距和对齐。

直线的 `x1`、`y1`、`x2`、`y2` 同样先转换为局部几何，再整体平移到选中的候选位置。

## 架构

### `shortcutPlacementManager.ts`

在 `src/engine/managers/shortcutPlacementManager.ts` 新增菜单快捷落位管理器。它只负责计算，不创建 Fabric 元素、不写历史记录，也不直接修改 Pinia store。

输入包括：

- 设计宽高、中心点和圆形安全区。
- 当前画布内容元素的类型与边界。
- 待添加单元素或组合块的类型、估算尺寸和成员相对位置。
- `smart` 或 `fixedCenter` 落位模式。

输出包括：

- 布局块的最终锚点或整体位移。
- 选中候选的边界和评分结果，供开发调试使用。

主要职责拆分为纯函数：

- 收集和标准化已有元素边界。
- 根据元素配置估算待添加块的占用尺寸。
- 按类型生成候选位置。
- 校验圆形安全区。
- 计算交叠、类型区域、距离和对齐评分。
- 用真实边界执行最多一次复核落位。

### `AppMenu.vue`

`AppMenu.vue` 继续负责：

- 组装元素配置和数据属性。
- 加载字体。
- 调用元素注册器创建 Fabric 对象。
- 显示成功或失败提示。
- 提交最终历史状态。

原有单元素处理函数改为调用统一的“快捷布局块”入口。单元素被表示为只有一个成员的布局块；数据字段和 Goal 快捷项一次性提交多个成员。这样单元素和组合元素共享相同的落位、回滚和历史记录规则。

## 候选评分

先淘汰不满足安全区约束的候选，再按以下字段组成的元组进行字典序比较，避免依赖难以维护的魔法权重：

1. **安全区约束**：候选块完整处于圆形安全区内才可进入正常评分。矩形块以四个角检查，直线以端点和实际描边边界检查。
2. **重叠面积**：计算候选块与所有非结构内容元素的交叠面积。使用按新块面积归一化的交叠比例作为主要惩罚。计算前将已有边界按标准 `454 × 454` 画布上的 8 像素向外扩展，实际间距乘以 `min(width, height) / 454`，避免视觉上贴得过近。
3. **类型区域优先级**：优先区域的候选优于同等重叠程度的后续区域。
4. **与首选锚点的距离**：在同一区域内选择移动距离较短的位置。
5. **同类和关联元素对齐**：时间与日期优先共享水平中心；重复状态图标优先形成水平或垂直序列；数据块优先形成规则行列。
6. **固定候选序号**：所有其他分数相同时，用候选生成顺序决定结果。

比较元组为 `[归一化重叠比例, 区域序号, 首选锚点距离, 对齐惩罚, 候选序号]`。只要存在零重叠候选，就不选择有重叠候选。若所有候选都重叠，则从整个安全区候选中选择重叠面积最小者；重叠相同再比较类型区域和距离。添加不能因为画布拥挤而弹出确认框或被阻止。

## 尺寸估算与真实边界复核

创建前根据配置估算占用尺寸：

- 有 `width`、`height`、`radius` 或直线端点的元素直接使用几何配置和描边宽度。
- 文字、时间、日期、数据和图标使用 `fontSize`、格式预览文本和既有字体度量约定估算。
- 组合块取所有成员局部边界的联合矩形。

先用估算尺寸选择候选并写入创建配置，因此普通元素不会先默认出现在中心。元素创建后读取返回 Fabric 对象的 `getBoundingRect()` 真实边界：

- 若真实边界仍在安全区内，且没有更优的可避免重叠，保留初始位置。
- 若发生超界或出现可避免重叠，用真实尺寸重新评分一次并整体平移；复核评分必须从已有占用列表中排除本次新建块的全部成员。
- 最多复核一次，防止多轮移动和不可预测抖动。

最终移动必须同时同步 Fabric 坐标、`setCoords()` 和元素的持久化配置，不能只改变画面上的对象。

## 添加事务与历史记录

一次菜单快捷添加是一个原子操作：

1. 保存本次开始前的数据属性键集合。
2. 创建所需数据属性并组装单元素或组合块配置。
3. 读取当前画布边界并计算初始落位。
4. 在 `historyStore.runWithoutRecording()` 中创建全部成员。
5. 使用真实 Fabric 边界执行一次复核和最终同步。
6. 请求画布重绘，并通过 `historyStore.saveState('shortcut:add', { captureConfig: true })` 只提交一个带最终配置的历史快照。

因此数据字段或 Goal 组合一次撤销即可整体删除，不会把临时初始位置或各子元素分别写成多个撤销步骤。

## 异常处理

- 画布尚未初始化或边界数据不可用：使用该类型的首选锚点并限制在安全区内，不统一回退到中心。
- 单个已有 Fabric 对象无法提供有效边界：忽略该对象的碰撞边界，继续计算，并在开发控制台记录警告。
- 智能评分出现异常：使用类型首选锚点完成添加；轴心元素仍使用设计中心。
- 组合中任一成员创建失败：移除本次已创建的其他成员，删除本次新建且此前不存在的数据属性，不提交历史快照，并沿用现有错误提示。
- 真实边界复核失败：保留已经通过安全区检查的初始落位，不进行多次重试。

## 验证与验收

当前工作区正在接入 Vitest（`npm run test:unit`）。落位管理器保持纯计算边界，并在 `src/engine/managers/shortcutPlacementManager.test.ts` 添加表驱动测试，至少覆盖：

- 轴心类型固定返回设计中心。
- 各普通类型优先选择对应区域。
- 已有元素占用首选位置时选择零重叠候选。
- 所有候选都被占用时稳定选择重叠面积最小者。
- 候选不会越出圆形安全区。
- 同一输入重复计算得到相同结果。
- 组合块整体平移后成员相对位置不变。

本次执行：

- `npm run test:unit`，运行落位管理器及仓库已有单元测试。
- `npm run build`，覆盖 `vue-tsc --noEmit` 和生产 Vite 构建。
- `git diff --check`，检查新增和修改文件格式。

人工验收场景：

1. 空画布依次添加时间、日期、状态、天气、数据字段、图表和形状，各类型进入预期优先区域。
2. 连续添加多个同类元素，元素按固定顺序避让且不重叠；刷新后以相同初始状态重做，结果一致。
3. 在已有混合内容的画布中添加元素，优先区被占用后选择附近空位。
4. 数据字段、Goal Bar 和 Goal Arc 保持内部相对布局，并且一次撤销删除整个组合。
5. 在接近满载的画布中添加元素，仍能完成添加，并选择圆形安全区内重叠面积最小的位置。
6. 添加时针、分针、秒针和中心盖，始终位于设计中心。
7. 在非 `454 × 454` 设计尺寸上重复上述测试，候选区域随设计尺寸缩放。
8. 保存并重新进入设计，智能落位后的坐标、组合结构和图层顺序保持一致。
9. 确认左侧 Add Element、复制粘贴、导入和手动拖动行为没有变化。
10. 确认实现没有覆盖工作区中已有的 Goal 和颜色选择器未提交改动。

## 成功标准

- 顶部菜单快捷添加的普通元素不再全部堆叠在设计中心。
- 类型语义、自动避让和圆形安全区规则同时生效。
- 组合元素完整落位并作为一次历史操作处理。
- 拥挤画布存在确定性的最小重叠兜底结果。
- 构建通过，保存和重新加载后位置不漂移，任务范围之外的添加流程无行为变化。
