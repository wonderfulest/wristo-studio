# 左侧图层分组折叠设计

## 目标

减少同一数据分组在左侧图层面板中占用的空间，让用户先看到清晰的图层结构，需要精细编辑时再展开组内元素。

## 范围

本次只调整 `wristo-studio` 左侧图层面板的展示与交互。分组摘要行是面板生成的虚拟节点，不创建 Fabric 对象，不修改设计配置结构、`orderIds`、导出格式或设备端运行逻辑。

`background` 和 `global` 是固定层，不参与自动分组。

## 分组规则

- 沿用 `LayerPanel.vue` 当前的分组键解析顺序：`groupId`、`groupKey`、`groupName`、`parentId`、`dataProperty`、`goalProperty`。
- 具有相同非空分组键且成员数不少于 2 的可移动图层组成一个分组。
- 单成员或没有分组键的图层继续按普通图层展示。
- 分组成员在面板中应保持连续；现有整组拖拽时保持成员相对顺序的约束继续生效。

## 展示

- 分组默认折叠，只显示一条虚拟摘要行。
- 摘要行显示展开箭头、分组名称和成员数量，例如 `Steps · 3 项`。
- 展开后，摘要行保留在顶部，真实成员图层显示在其下方。
- 成员行继续提供现有的选中、高亮、显隐、锁定和删除操作。
- 摘要行不显示容易产生歧义的单成员删除、显隐和锁定按钮。
- 展开状态仅保存在当前编辑会话中；重新进入设计时所有分组恢复默认折叠。

## 交互

### 展开与收起

- 点击摘要行的箭头或双击摘要行，在展开与收起之间切换。
- 单击摘要行主体选中组内全部可选成员，不触发展开。
- 当用户直接从画布选中一个属于折叠组的元素时，对应分组自动展开，使被选中的真实图层可见。

### 拖拽

- 折叠时拖拽摘要行，整组作为一个连续单元移动。
- 展开时拖拽摘要行，仍移动整组并保持组内相对顺序。
- 展开后的成员行维持当前行为；拖动其中一个成员时，仍按现有规则移动整个分组，避免成员被拆散。
- 面板使用“顶部为画布最上层”的顺序；提交给 `applyOrder()` 前继续转换为 Fabric 和 `orderIds` 使用的“底部到顶部”顺序。

### 选择

- 摘要行选中整组时，通过画布已有多选能力激活所有可选择成员。
- 已锁定或不可选择的成员不应阻止其余成员被选中。
- 如果组内没有可选择成员，摘要行保持可展开，但点击主体不改变画布选择。

## 状态与数据流

- `LayerPanel.vue` 持有当前会话的展开分组键集合，初始为空。
- 一个纯函数根据顶部到下方的面板图层顺序和展开集合生成渲染节点：普通图层节点、分组摘要节点以及展开时的成员节点。
- 拖拽不能直接把虚拟摘要节点 ID 写入 `orderIds`。拖拽结束后先将每个摘要节点展开为真实成员 ID，再复用现有连续分组规范化与画布顺序转换逻辑。
- 图层同步、删除或设计切换后，展开集合中已不存在的分组键应被清理。

## 异常与边界

- 分组名称为空时不会形成分组摘要。
- 分组从多个成员减少到一个成员后，摘要行消失，剩余成员恢复为普通图层。
- 分组成员新增后，成员数量立即更新，但当前会话中的展开状态保持不变。
- 折叠只隐藏面板成员行，不改变成员在画布上的可见性、锁定状态或选择状态。

## 测试与验收

至少覆盖以下自动化测试：

1. 同组两个及以上图层默认生成一条折叠摘要。
2. 展开后摘要和全部真实成员按原顺序显示。
3. 无组图层和单成员分组保持普通图层。
4. 折叠摘要转换为拖拽顺序时只产生真实成员 ID，且保持组内相对顺序。
5. 面板顶部到画布底部的顺序转换仍然正确。
6. 成员删除导致分组只剩一个时自动取消摘要。
7. 画布选中折叠组内成员时自动展开对应分组。

完成后运行相关 Vitest 用例、完整 `npm run test:unit`、`npm run build` 和 `git diff --check`。

## 非目标

- 不持久化展开/收起状态。
- 不提供分组重命名、创建、解散或跨组移动能力。
- 不给摘要行增加整组删除、显隐或锁定按钮。
- 不修改 Fabric 对象模型、后端设计协议或 Connect IQ 生成逻辑。
