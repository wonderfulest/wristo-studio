# 图表控制点尺寸调整设计

## 目标

让 `barChart` 和 `lineChart` 通过画布上的八个控制点直接调整尺寸，并将设置面板中的宽度、高度改为只读显示。保存和导出时继续使用实际 `width`、`height`，不把 Fabric 临时缩放值作为图表尺寸契约。

## 功能范围

- `barChart` 和 `lineChart` 启用八个控制点。
- 四角控制点等比例缩放宽高。
- 上下控制点只调整高度，左右控制点只调整宽度。
- 拖拽结束后将 `scaleX`、`scaleY` 合并到实际宽高，并重置为 `1`。
- 图表内容按新宽高重新绘制，位置中心保持不变。
- 设置面板继续显示宽度和高度，但用户不能手动输入。
- 面板中的宽高随画布调整结果同步刷新。

不在本次范围内：修改其他元素的控制点、改变图表数据或样式设置、增加锁定宽高比开关。

## 控制点与尺寸规则

两个图表复用 `controlManager` 已有的 `resize8` 控制模式，并加入统一控制点适用类型列表。创建或恢复图表对象时设置：

```ts
hasControls: true
designerControlMode: 'resize8'
```

尺寸限制沿用当前面板已有规则：

| 元素 | 最小宽度 | 最大宽度 | 最小高度 | 最大高度 |
| --- | ---: | ---: | ---: | ---: |
| `barChart` | 60 | 454 | 20 | 227 |
| `lineChart` | 50 | 454 | 20 | 227 |

拖拽得到的目标宽高按上述范围限制。若某一方向触发限制，只限制该方向；四角等比例拖拽仍由 Fabric 维持比例，但归一化后的最终尺寸必须落在允许范围内。

## 尺寸归一化与重绘

图表尺寸的来源保持为 `width` 和 `height`。控制点拖拽过程中 Fabric 临时改变 `scaleX`、`scaleY`；拖拽结束后执行以下流程：

1. 读取 `width * scaleX` 和 `height * scaleY` 得到显示尺寸。
2. 对显示尺寸应用图表类型对应的最小值和最大值。
3. 将结果写回 `width`、`height`，并把 `scaleX`、`scaleY` 重置为 `1`。
4. 使用现有图表重绘函数重建透明命中矩形、柱体或折线与控制点。
5. 恢复原中心位置，更新坐标并刷新画布。

归一化处理必须避免重复绑定对象事件。创建、恢复、更新图表后都应保留 `resize8` 控制模式，使全局控制管理器可以稳定重新应用控制点。

## 设置面板同步

宽度和高度仍使用现有数字输入组件和单位显示，但设置为禁用或等效只读状态，并移除其修改事件。其他图表设置保持可编辑。

面板不能只在切换选中元素时同步；当前选中图表通过控制点调整后，也必须从元素或编码配置收到更新，显示归一化后的实际宽高。

## 保存与兼容性

编码器输出归一化后的实际 `width`、`height`。为防止极短时间内在 Fabric 修改事件与归一化回调之间编码，编码器也应按显示尺寸处理非 `1` 的缩放值，避免保存旧宽高。

旧设计不需要迁移：载入后仍从已有 `width`、`height` 创建图表，缩放默认为 `1`。其他样式、数据字段和位置字段保持原格式。

## 测试与验证

- 测试两个图表创建后启用 `resize8` 和控制点。
- 测试水平、垂直和等比例缩放可归一化为实际宽高，且缩放值重置为 `1`。
- 测试最小、最大宽高限制。
- 测试编码器在存在临时缩放值时仍输出显示尺寸。
- 测试设置面板的宽高输入不可编辑，并且不再发出宽高修改。
- 运行相关 Vitest 测试、`npm run build` 和 `git diff --check`。
