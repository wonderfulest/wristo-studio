# 颜色属性 RGB565 默认颜色设计

## 背景

Studio 的颜色属性目前把默认颜色限制在已有颜色选项中。新建属性时，如果最近使用的颜色不在预设 64 色内，还会自动新增一个 `Custom #RRGGBB` 候选项。这使“属性默认颜色”和“用户可切换的颜色集合”共用同一限制，无法满足 Garmin RGB565 默认颜色的完整色域，同时也允许颜色属性候选项超出预设集合。

## 目标

- 颜色属性的默认颜色支持完整 RGB565 色域，共 65,536 个值。
- Studio 颜色选择器继续使用直观的 `#RRGGBB` 输入和预览。
- 保存的默认颜色必须先量化为 RGB565，再以 `0xRRGGBB` 形式持久化。
- 颜色属性的可切换候选项固定为 `Default`、现有 64 种标准颜色和 `Transparent`。
- 不修改生成模板和 Monkey C runtime 的颜色解析协议。

## 非目标

- 不允许用户为颜色属性添加任意自定义候选颜色。
- 不把颜色属性的设置界面改为 Garmin 原生自由颜色选择器。
- 不增加新的 runtime 哨兵值，例如 `-2`。
- 不改变普通元素颜色控件的行为。

## 数据设计

### RGB565 量化

颜色选择器返回 24 位 RGB 值。保存前分别量化红、绿、蓝通道：

- 红色保留 5 位，映射到 0 至 31。
- 绿色保留 6 位，映射到 0 至 63。
- 蓝色保留 5 位，映射到 0 至 31。
- 再将量化通道展开回 8 位显示值，生成稳定的 `0xRRGGBB` 字符串。

同一 RGB565 颜色无论经过多少次打开和保存，都必须得到相同结果，避免颜色值漂移。

### 固定候选集合

颜色属性的 `options` 按以下顺序生成：

1. `Default`，value 为该属性量化后的默认颜色。
2. 现有 64 种标准颜色，保持当前顺序、标签和值不变；若某项与 `Default` 数值相同，则过滤该标准色项。
3. `Transparent`，value 为 `-1`。

`Default` 不是特殊哨兵值。它直接使用默认颜色的数值，因此现有 `properties.j2.xml`、`settings.j2.xml` 和 Monkey C 读取逻辑可以继续把它当作普通 number 处理。

`Default` 始终保留。若默认颜色与某个标准色相同，则过滤对应标准色项，确保 Connect IQ 的同一 list setting 中不存在重复 value。

## Studio 交互

### 默认颜色

`Default Value` 区域继续使用现有颜色选择器。用户可选择任意 `#RRGGBB`；选择完成后，界面立即显示量化后的 RGB565 结果，并同步更新 `Default` 候选项的 value。

默认颜色不必出现在 64 种标准色中，也不会生成 `Custom #RRGGBB` 项。

### 颜色候选项

颜色候选项区域改为只读展示固定集合：

- 移除 Add Option。
- 移除标签和值的输入框。
- 移除排序和删除按钮。
- 保留色块、名称和值，便于确认最终导出的设置列表。

表单提交前重新根据默认颜色构建固定候选集合，不信任界面中的临时数组，防止异常状态或旧数据把自定义颜色带入导出配置。

## 旧数据兼容

编辑已有颜色属性时：

- 读取原有 `value` 作为默认颜色；若是合法颜色，则量化为 RGB565。
- 若原有默认值为 `-1`，回退为白色 RGB565 默认色；透明色只能作为可切换候选项，不能作为新的默认颜色。
- 丢弃旧 `options` 中的自定义项、重复项和异常项。
- 重新生成 `Default + 不重复的标准色 + Transparent` 固定集合。

旧设计只在用户打开并确认颜色属性对话框后完成规范化；加载设计时不做全局静默迁移，避免无关编辑改变已有文件。

## 边界与错误处理

- 空值、格式错误或无法解析的默认颜色回退为 `0xFFFFFF`。
- 颜色解析兼容 `#RRGGBB`、`0xRRGGBB` 和纯六位十六进制输入。
- `Transparent` 的内部值继续使用 `-1`。
- 表单验证确保默认值是合法、已量化的 RGB565 展开值，并确保 options 与固定集合一致。

## 测试与验证

- 单元测试 RGB565 量化：黑、白、各通道边界、非色域输入和幂等性。
- 单元测试固定候选集合：顺序、Default 同步、非重复标准色保持不变、与 Default 相同的标准色被过滤、Transparent 位于末尾。
- 对话框测试：自由选择默认颜色后得到量化值，不新增 Custom 项，候选项不可编辑。
- 兼容测试：编辑含自定义 option 的旧属性后，只保存固定集合。
- 运行相关 Vitest 测试、`npm run build` 和 `git diff --check`。
