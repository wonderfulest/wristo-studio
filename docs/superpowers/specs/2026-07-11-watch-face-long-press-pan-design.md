# Studio 表盘区域长按拖动画布设计

## 背景

Studio 已支持在表盘实际轮廓外按下后立即拖动整个画布舞台，Fabric 内容、表盘底层和设备外框会作为一个整体移动。当前表盘区域内的指针事件全部交给 Fabric，因此在表盘空白处或元素上按住时，只能执行选择、框选、移动、缩放和旋转，不能平移整个画布。

本功能在不破坏现有 Fabric 编辑行为的前提下，为表盘区域增加长按平移。表盘外部的立即平移保持不变。

## 目标

- 在表盘区域内按住 400ms 后，可以拖动整个画布舞台。
- 长按可以从表盘空白处、元素本体或 Fabric 控制点开始。
- 400ms 内移动超过 6px 时，取消长按识别并继续原有 Fabric 拖动。
- 400ms 内松开时，保留原有点击、选择和控制点行为。
- 长按接管画布平移时，不移动或修改元素，不产生 `object:modified`，不写入撤销历史。
- 表盘外部仍然按下立即平移，不增加等待时间。
- 继续复用现有舞台偏移、边界限制、标尺刷新和 Fabric `calcOffset()` 同步逻辑。

## 非目标

- 不增加空格键、鼠标中键或单独的平移工具。
- 不增加长按进度动画、震动反馈或提示文案。
- 不改变元素拖动、框选、缩放、旋转、双击编辑和多选规则。
- 不持久化画布偏移，不改变设计配置、导出格式或自动保存内容。
- 不修改 Fabric 源码或替换其事件系统。

## 方案选择

### 采用：允许原生按下，并在长按窗口拦截轻微移动

表盘区域内收到主指针按下时，Studio 先创建一个等待中的长按会话，但不阻止 `pointerdown` 继续传给 Fabric。这样 Fabric 可以照常完成对象命中和选择。

在 400ms 等待窗口内，Studio 在捕获阶段拦截不超过 6px 的 `pointermove`，防止 Fabric 因手部抖动而移动元素。如果移动距离超过 6px，立即取消长按会话并放行当前及后续移动；Fabric 已收到最初的按下事件，可以直接根据当前位置继续原生拖动。

长按到期时，Studio 先通过集中封装的 Fabric 交互收尾适配器结束当前按下交互，再建立 Pointer Capture 并进入现有画布平移会话。因为等待窗口内没有移动事件进入 Fabric，收尾不会产生元素修改事件。

Fabric 6.7 默认分别监听 Mouse Events 和 Touch Events，而 Studio 的协调层使用 Pointer Events。初始化 Canvas 时必须显式启用 Fabric 官方的 `enablePointerEvents` 选项，使按下、移动和松开使用同一事件链；否则仅拦截 `pointermove` 不能阻止兼容 `mousemove` 或 `touchmove` 提前修改元素。

该方案的优点是保留真实的 Fabric 按下事件，不需要重建复杂的选择、控制点和双击语义，同时能阻止长按抖动造成的元素位移。

### 未采用：延迟并重放 Fabric 指针事件

该方案会在 400ms 内完全阻止 Fabric，再在识别为元素拖动时合成并重放按下和移动事件。合成事件难以完整还原控制点、双击、多选和浏览器可信事件属性，维护风险较高。

### 未采用：专用平移模式或快捷键

专用模式实现简单，但不能满足直接在表盘区域长按拖动的交互目标，并会增加额外模式状态和学习成本。

## 交互状态机

交互状态分为 `idle`、`pending-long-press` 和 `panning`：

1. `idle` 状态下，表盘外部的主指针左键按下立即进入 `panning`。
2. `idle` 状态下，表盘区域内的主指针左键按下进入 `pending-long-press`，并启动 400ms 定时器；原始按下事件继续传给 Fabric。
3. `pending-long-press` 状态下：
   - 当前指针相对按下起点的屏幕直线距离不超过 6px：拦截移动事件，并更新最后指针位置。
   - 当前指针相对按下起点的屏幕直线距离大于 6px：清除定时器，回到 `idle`，放行本次移动，后续由 Fabric 原生处理。
   - 400ms 内正常松开：清除定时器，回到 `idle`，放行松开事件，保留点击或选择行为。
   - 收到 `pointercancel`：清除定时器并收尾 Fabric 按下交互，不启动平移。
   - 定时器到期：收尾 Fabric 按下交互，以最后指针位置为平移起点，进入 `panning`。
4. `panning` 状态下，复用现有舞台偏移和边界限制逻辑；`pointerup`、`pointercancel` 或 `lostpointercapture` 结束平移并回到 `idle`。

距离判断使用屏幕 CSS 像素和平方距离。恰好 6px 仍属于长按容差，只有大于 6px 才切换回 Fabric 拖动。长按生效时以最后指针位置作为平移起点，避免舞台发生最多 6px 的跳动。

## 命中与优先级

指针按下位置分为三类：

- 编辑器控件：缩放、历史、时间模拟器、标尺和面板调整手柄等现有排除区域不参与画布平移。
- 立即平移区域：没有 Fabric 命中目标且位于表盘实际轮廓外的区域，保持按下立即平移。
- 长按候选区域：位于表盘实际轮廓内，或命中 Fabric 元素/控制点的位置。

圆形表盘透明角落默认属于立即平移区域；如果该位置命中延伸到表盘轮廓外的 Fabric 元素或控制点，则归入长按候选区域。这样快速拖动仍然操作元素，长按才接管整个画布。

仅主指针和左键参与识别。非主触点、右键和中键保持现有行为。工作区继续使用 `touch-action: none`，避免触控长按期间触发页面滚动。

## 组件职责

### `canvasManager.ts`

- 初始化 Fabric Canvas 时设置 `enablePointerEvents: true`，统一鼠标、触控笔和主触点的事件链。
- 不改变 Fabric 的对象选择、控制点或 transform 实现。

### `Design.vue`

- 将当前“是否可立即平移”的布尔判断拆分为排除、立即平移和长按候选三种结果。
- 持有等待中的长按会话、400ms 定时器和最后指针位置。
- 等待长按时在 document 的 `pointermove` 捕获阶段执行 6px 容差判断，即使指针移出工作区也能先于 Fabric 的 document 级冒泡监听器处理。
- 统一启动和结束画布平移，并根据来源决定是否清除元素选择。
- 表盘外部立即平移时保持现有清除选择行为；表盘内部长按接管时保留按下后形成的选择状态。
- 在组件卸载时清除长按定时器、Pointer Capture 和动画帧。

### `Canvas.vue`

- 暴露一个只用于舞台平移接管的 Fabric 交互收尾方法。
- 该方法先确认当前 transform 尚未执行；如果 transform 存在，则调用 `endCurrentTransform()` 完成无修改收尾并清除原控制点标记。
- 随后使用一个位于 Canvas 边界外的取消事件调用 Fabric 6.7 指针收尾路径，清理框选状态和 document 级 move/up 监听器。边界外事件不会命中原控制点，因此长按克隆、删除等自定义控制点时不会意外执行其 `mouseUpHandler`。
- Fabric 私有方法访问集中在此适配器中，并带有存在性检查；`Design.vue` 不直接读取或修改 Fabric 私有字段。
- 适配器的前置条件是等待期间的移动事件尚未进入 Fabric，因此不会触发 `object:modified`。

### `canvasPan.ts`

- 保留现有表盘轮廓命中和偏移边界函数。
- 增加独立的长按移动容差判断函数，使用平方距离避免不必要的开方。
- 纯函数不依赖 Vue、DOM 或 Fabric，便于覆盖 6px 边界测试。

## 数据与事件流

1. `Design.vue` 捕获 `pointerdown` 并完成区域分类。
2. 立即平移区域直接复用现有 `canvasPanSession`。
3. 长按候选区域创建包含 pointer ID、起点、最后位置、原始按下事件和定时器 ID 的临时会话，然后让事件继续到 Fabric。
4. 等待期间的轻微移动由 `Design.vue` 截止在捕获阶段；超过容差的移动会取消临时会话并继续传播。
5. 长按到期后，`Canvas.vue` 完成 Fabric 交互收尾，`Design.vue` 获取 Pointer Capture 并创建正式平移会话。
6. 平移帧继续更新 `panOffset`，刷新标尺并同步 Fabric DOM 偏移。
7. 平移结束后只清理瞬时状态，不进入 Pinia、设计数据或历史记录。

## 选择与历史行为

- 表盘外部立即平移沿用现有逻辑：开始平移时清除当前选择。
- 元素上长按时，Fabric 在按下阶段形成或保留的选择继续显示，平移结束后仍可编辑该元素。
- 表盘空白处长按时，Fabric 原本在按下阶段清除选择的行为保持不变。
- 等待期间不向 Fabric 传播轻微移动，因此元素 transform 的 `actionPerformed` 保持为 `false`。
- 长按接管前运行完整的 Fabric 指针收尾，清除其 document 监听；不直接调用只结束 transform、但不清理监听器的半完成路径。
- 画布平移不触发 `object:modified`，因此 `historyStore` 不新增记录。

## 异常与清理

- 新的主指针按下前先清理已失效的等待会话，避免残留定时器接管后续交互。
- `pointercancel` 在等待阶段只收尾 Fabric，不启动平移；在平移阶段使用现有结束逻辑。
- `lostpointercapture` 和显式结束可以重复调用，清理逻辑必须幂等。
- Fabric 尚未初始化或收尾适配器不可用时，取消该次内部长按接管；表盘外部的舞台平移仍可工作。
- 组件卸载时清除定时器，不让延迟回调访问已销毁的 Canvas。

## 验收与验证

- 表盘外部按下后仍立即拖动，无 400ms 延迟。
- 表盘空白处按住 400ms 后，整体画布跟随拖动。
- 元素本体上按住 400ms 后，整体画布跟随拖动，元素坐标不变且选择状态保留。
- Fabric 缩放、旋转或其他控制点上按住 400ms 后，可以拖动整体画布，元素属性不变。
- 在 400ms 内移动超过 6px，元素从原按下位置开始正常拖动，不需要重新按下。
- 在 400ms 内移动恰好 6px，仍可进入长按平移；超过 6px 才取消。
- 在 400ms 内松开，点击、选择、清空选择、双击编辑和控制点点击行为保持不变。
- 长按等待期间的轻微手抖不会移动元素。
- 长按平移开始和结束时画布无位置跳变，指针移出工作区后松开仍能正常结束。
- 圆形表盘透明角落仍立即平移；命中延伸元素或控制点时改为长按候选。
- 长按平移不新增撤销历史，保存和导出结果不包含舞台偏移。
- 扩展 `tests/canvasPan.test.mjs` 覆盖 6px 容差边界，并运行 `npm run test:canvas-pan`。
- 运行 `npm run test:canvas-pan`、`npm run typecheck` 和 `npm run build` 验证画布手势、类型和生产构建。
