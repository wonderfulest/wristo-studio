# Studio Sub-dial Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 Wristo Studio、素材包、Connect IQ 脚手架和 SuperBarrel 运行时中实现可绑定单个数据字段、支持百分比/自定义范围及自定义图片指针的多功能小表盘元素。

**Architecture:** Studio 将 `subDial` 实现为一个专用 Fabric Group，并通过纯计算模块统一数值归一化和角度规则；renderer 只在结构变化时重排静态子对象，数据变化只更新指针与文本。导出链路把扁平配置和原始指针素材送入脚手架，脚手架生成设备 PNG 资源，SuperBarrel 的 `SubDial.mc` 负责设备端静态缓存与动态绘制。

**Tech Stack:** Vue 3、TypeScript、Fabric.js 6、Pinia、Vitest、JSZip、Python 3、Bash、Jinja、Garmin Monkey C / Connect IQ SDK。

```ts
export function normalizeSubDialValue(
  value: number,
  minValue: number,
  maxValue: number,
  behavior: 'clamp' | 'hide',
): number | null {
  if (![value, minValue, maxValue].every(Number.isFinite) || maxValue <= minValue) return null
  const progress = (value - minValue) / (maxValue - minValue)
  if (behavior === 'hide' && (progress < 0 || progress > 1)) return null
  return Math.max(0, Math.min(1, progress))
}

export function resolveSubDialAngle(
  progress: number,
  startAngle: number,
  endAngle: number,
  counterClockwise: boolean,
  rotationOffset: number,
): number {
  let sweep = endAngle - startAngle
  if (counterClockwise && sweep > 0) sweep -= 360
  if (!counterClockwise && sweep < 0) sweep += 360
  return startAngle + sweep * progress + rotationOffset
}

export const clampPivot = (value: number) => Math.max(0, Math.min(1, Number.isFinite(value) ? value : 0.5))
```

```monkeyc
module SubDial {
    function normalize(value as Number, minValue as Number, maxValue as Number, hideOutOfRange as Boolean) as Number or Null;
    function angleFor(progress as Number, startAngle as Number, endAngle as Number, counterClockwise as Boolean, offset as Number) as Number;
```

- [ ] **Step 2: 实现归一化和显式数值类型转换**

- [ ] **Step 3: 实现静态缓存和动态绘制**

cache Dictionary 使用元素 ID 保存背景/刻度 Bitmap 和已加载指针 Bitmap。`draw()` 首次建立静态缓存，随后只重绘值、单位和指针。Line/Triangle 走矢量绘制；Image 根据相对 pivot 计算旋转后位置。设备不支持需要的旋转 API 时，使用项目现有兼容绘制工具或生成阶段角度帧，不允许静默画错轴心。
